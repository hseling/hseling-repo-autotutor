Autotutor


Сервис выполнен в ходе научно-исследовательского семинара магистерской программы НИУ ВШЭ «Компьютерная Лингвистика».

В рамках проекта исследовательская группа занималась разработкой алгоритма для рекомендации текстов русского языка для студентов-иностранцев. Основная идеея проекта — отказаться от стандартных CEFR-подобных метрик сложности текста и использовать более подробную декомпозицию языковых явлений применительно к знаниям студента.

В реализованном сервисе пользователю предлагается прочитать текст и выполнить тест не менее четырёх раз, после чего пользователь получает набор текстов, рекомендованных алгоритмом. В процессе накопления информации о знаниях пользователя (то есть с увеличением количества пройденных тестов) алгоритм будет выдавать более точные рекомендации.

Источник текстов: научно-популярное издание ТАСС «Чердак», закрытое в 2019 году. Сейчас используемые нами и им подобные тексты можно найти на сайте nauka.tass.ru. За предоставленные тексты благодарим команду проекта Анализатор научно-популярных текстов.


Краткое описание алгоритма

В процессе предобработки текст разбивается на совокупность векторов трех уровней: уровня слова, предложения и текста. Векторы слов — это предобученные word2vec-эмбеддинги; векторы предложений — это совокупность значений, соответствующих различным грамматическим явлениям, содержащимся в предложении, которые представляют сложность для студентов-иностранцев; векторы текстов — это общетекстовые метрики сложности типа Type Token Ratio (отношение уникальных лексем ко всем токенам), lix и других.

Проходя тесты, студент отвечает на вопросы о предложениях из текста: понятно ли ему то или иное предложение. Отвечая на вопросы, студент производит действие, схожее с разметкой датасета, и в течение прохождения нескольких текстов (от четырёх, больше — лучше) создаются три размеченных датасета (векторы текстов, предложений и слов размечаются как подходящие и не подходящие студенту). Эти датасеты отображают знания студента. Далее используется алгоритм ближайших соседей, чтобы из предобработанного набора текстов подобрать такие тексты, чьи составные векторы наиболее близки к положительно размеченным векторам из соответствующего датасета студента, но с небольшим зазором, чтобы текст не оказался слишком прост.
